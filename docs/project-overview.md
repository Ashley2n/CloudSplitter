# Cloud Splitter API

## Purpose

Cloud Splitter is a backend-focused software engineering project designed to help groups of people track shared expenses and calculate how much each participant owes after expenses are divided.

The primary goal of this project is not to create a production-ready financial application. The goal is to develop professional software engineering skills through the implementation of a realistic business problem.

The project emphasizes:

* Test-Driven Development (TDD)
* API design
* Domain modeling
* Database design
* Repository patterns
* Service-layer architecture
* Automated testing
* Software documentation

---

## Problem Statement

When multiple people participate in a shared activity such as:

* Vacations
* Roommate living arrangements
* Events
* Group dinners
* Business travel

Expenses are often paid by different individuals throughout the activity.

Determining who owes money to whom becomes increasingly difficult as the number of participants and expenses grows.

Cloud Splitter provides a centralized system that records:

* Groups
* Participants
* Expenses
* Payments

The system then calculates balances and settlements automatically.

## Core Features

### Group Management

Users can:

* Create groups
* View groups
* Update groups
* Delete groups

### Participant Management

Users can:

* Add participants
* Remove participants
* View participants

### Expense Management

Users can:

* Create expenses
* Assign a payer
* Record amounts
* Categorize expenses

### Balance Calculations

The system calculates:

* Total expenses
* Individual contributions
* Amount owed by each participant
* Settlement recommendations
